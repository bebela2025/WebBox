O QUE É BANCO DE DADOS

Um banco de dados é um sistema estruturado para armazenar, gerenciar e manipular grandes volumes de informações.
Esses dados podem ser numéricos, textos, imagens, vídeos, entre outros, em qualquer suporte de informação digital.
A principal função de um banco de dados é garantir que essas informações sejam armazenadas de forma organizada e possam ser rapidamente manipuladas de forma segura e confiável.

Para facilitar o entendimento, podemos compará-lo a uma biblioteca pública. Veja a analogia:

Tabelas

são como estantes, organizadas por tema.

Linhas

representam os livros individuais, cada um contendo um conjunto único de informações.

Colunas

são as categorias: autor, título, editora, etc.

ESTRUTURA DE UM BANCO DE DADOS

Um banco de dados é composto por elementos que facilitam sua organização e o acesso às informações. Entre os principais, destacam-se:

Tabelas

Armazenam os dados em linhas e colunas.

Registros

Cada linha representa um conjunto completo de informações.

Campos

Cada coluna representa um tipo específico de dado (nome, idade, cidade).

Chaves

Chave primária

Identifica de forma única cada registro, evitando duplicações.

Chave estrangeira

Conecta informações entre diferentes tabelas, estabelecendo relacionamentos.

TIPOS DE BANCO DE DADOS

Banco de Dados Relacional (RDBMS)

usa tabelas com chaves e esquemas fixos. Ex: MySQL, Oracle, PostgreSQL.

Banco de Dados Não Relacional (NoSQL)

estrutura flexível, como documentos, grafos, colunas ou chave-valor. Ex: MongoDB, Redis, Cassandra.

Banco de Dados em Nuvem

operados por serviços como AWS, Google Cloud e Azure.

Banco de Dados Hierárquico

organiza dados em árvore (pai-filho), comum em sistemas antigos.

Banco de Dados em Rede

semelhante ao hierárquico, mas mais flexível, com múltiplas conexões.

FUNCIONALIDADES DE UM BANCO DE DADOS

BIG DATA

Big Data se refere ao grande volume de dados gerados constantemente. Ele é caracterizado por 5 Vs:

Volume

gigantesca quantidade de dados.

Velocidade

dados gerados em tempo real.

Variedade

informações de diversas fontes e formatos.

Veracidade

qualidade e confiabilidade dos dados.

Valor

extração de insights para conhecimento.

MODELO ENTIDADE-RELACIONAMENTO (MER)

Entidades

Representam objetos ou conceitos do mundo real que são relevantes para o sistema, como um "cliente", um "produto" ou um "pedido".

Classificação por Natureza:

Físicas

Objetos concretos e tangíveis (ex: pessoas, empresas, produtos)

Lógicas

Conceitos que existem a partir de interações no domínio de negócios (ex: Venda, Pedido)

Tipos Estruturais:

Fortes

Independentes (ex: Produto)

Fracas

Dependem de outra entidade (ex: Venda)

Associativas

Representam um relacionamento (ex: ItemVenda)

Relacionamentos

Indicam as associações ou interações entre as entidades. Por exemplo, um cliente pode "realizar" um pedido.

Tipos de Cardinalidade:

1..1 (Um para Um)

Cada entidade referencia apenas uma unidade da outra

1..n (Um para Muitos)

Uma entidade referencia várias unidades da outra, mas cada unidade do "muitos" está ligada a apenas uma do "um"

n..n (Muitos para Muitos)

Entidades de ambos os lados podem referenciar múltiplas unidades uma da outra

Atributos

São as características que descrevem cada entidade dentro do domínio. Por exemplo, um cliente possui nome, endereço e telefone.

Classificação por Função:

Descritivos

Característica intrínsecas (ex: nome, cor)

Nominativos

Identificam e definem um objeto (ex: código, número)

Referenciais

Ligam uma entidade a outra em um relacionamento (ex: CPF do cliente na entidade Venda)

Classificação por Estrutura:

Simples

Um único atributo (ex: nome, peso)

Compostos

Vários atributos formam uma informação (ex: endereço = rua + número + bairro)

Chave Primária

Atributo com valor único que identifica cada entidade de forma irrepetível (ex: CPF no cadastro de Cliente).

DIAGRAMA ENTIDADE-RELACIONAMENTO (DER)

Enquanto o MER é um modelo conceitual, o Diagrama Entidade Relacionamento (DER) é a sua representação gráfica:

Entidades

Representadas por retângulos

Relacionamentos

Representados por losangos

Atributos

Representados por elipses (ou dentro da entidade, na notação UML)

Cardinalidade

1:1, 1:N, N:N ou notações como (1,1), (0,n)
